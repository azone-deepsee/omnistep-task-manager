■ 1. 背景・相談の概要
--------------------------------------------------------------------------------
グループ単位でのタスク管理を検討する相談があった。
個人向けとして育ててきた PBPM の強み（計画と実績のタイムライン表示等）を
活かしつつ、チームでの共有・権限分離・永続化方式の見直しが求められている。

■ 4. 技術上の前提・制約（共有すべき点）
--------------------------------------------------------------------------------
4.1 ブラウザ単体の限界
  ・現在の HTML+JS をブラウザで開く構成では、任意フォルダの
    .accdb / .sqlite をページから直接読み書きできない（セキュリティ制約）
  ・「保存先パスを固定して起動時ロード」を厳密に実現するには、次のいずれかが必要：
      (a) ファイルを開く／名前を付けて保存（File API / File System Access API）
      (b) Electron 等のデスクトップアプリ化
      (c) ローカル常駐の薄い連携サーバー（Access / SQLite を中継）

4.2 Access を「一旦」使う場合の現実的な進め方
  ・Phase 1：共有フォルダ上のプロジェクトファイル（JSON 等）でグループ運用
  ・Phase 2：Access テーブルをマスターにし、エクスポート／インポートで OmniStep と同期
  ・Phase 3：SQLite 等へ同一スキーマで移行

4.3 同時編集
  ・小規模チームなら「最後に保存した人が勝つ」＋保存時の更新日時チェック警告
  ・本格的な同時編集が必要なら Phase 3 以降で検討


■ 5. 機能要件（将来版・案）
--------------------------------------------------------------------------------
5.1 タスクのスコープ（R03）
  ・タスク（または親テーマ）に scope を持たせる
      personal … 個人のみ
      team     … グループ共有
  ・ownerId … タスク主（6桁・既存設定を流用）
  ・teamId / sharedEditors[] … 共有メンバー（任意）
  ・UI：新規登録時に個人/チーム選択、フィルタ「個人のみ/チームのみ/すべて」

5.2 権限（R07）
  ロール          できること（案）
  --------------  ----------------------------------------------------------
  owner（主）     全項目編集・削除・共有設定
  editor（共有）  進捗・メモ・実績日・子タスク追加。親期間・削除は不可
  viewer（閲覧）  閲覧のみ（現行の外部CSV相当）

5.3 ファイル保存（R05, R06）
  ・設定項目：既定プロジェクトファイルパス（または前回開いたファイル）
  ・起動フロー：
      1. アプリ起動
      2. 設定の保存先からロード（または「ファイルを開く」）
      3. 作業（メモリ上のみ変更、isDirty 管理）
      4. 「保存」ボタンでファイルへ書き出し
      5. 終了時 … 未保存なら beforeunload 警告
  ・保存対象：タスク、年間日程、設定のうちプロジェクトに含める項目を定義
  ・自動保存 … オプション OFF をデフォルト（要望どおり）

5.4 データベース（R04）
  ・論理スキーマを RDB 想定で設計（Access / SQLite 共通）
      Projects, Tasks, TaskMembers, MasterEvents, AuditLog 等
  ・Access … 共有 .accdb ＋ 定期同期 or 連携ツール
  ・SQLite … 将来の軽量デスクトップ版・移行先

5.5 グループ共有（R01）
  短期案：共有フォルダ上の 1 プロジェクトファイル（.pbpm.json 等）
  中期案：Access マスター ＋ エクスポート／インポート
  長期案：ローカル API ＋ DB 直接 CRUD


■ 6. 段階的ロードマップ（案）
--------------------------------------------------------------------------------
  段階      版（案）   主な内容
  --------  ---------  ----------------------------------------------------
  Phase 1   1.5.x      手動保存・未保存警告・プロジェクトファイル開閉
                       （JSON。Access なしでもグループ試運用可能）
  Phase 2   1.6.x      scope（個人/チーム）・共有メンバー・権限UI
  Phase 3   2.0.x      保存パス固定（Electron 等）・Access 連携
  Phase 4   2.x        SQLite 移行・変更履歴・競合検知強化

  ※ R02（タイムライン）は Phase 1 以前の 1.1.4〜1.2.0 で概ね充足。
    グループ化時は担当者表示・フィルタ拡張を Phase 2 で検討。


■ 7. 確認事項（着手前にヒアリング）
--------------------------------------------------------------------------------
  1. 同時に何人が編集するか（1人ずつ保存 vs 同時編集必須）
  2. Access を誰が触るか（全員必須か、OmniStep のみのメンバーがいるか）
  3. 共有単位（1プロジェクト1ファイルか、チームごと複数か）
  4. デスクトップアプリ化（Electron）の可否（保存パス固定に直結）
  5. 「書き加え」の範囲（メモのみ / 進捗+子追加 / 実績日まで 等）
  6. 既存 localStorage データの移行方針（初回のみエクスポート等）


■ 8. 非機能要件（案）
--------------------------------------------------------------------------------
  ・後方互換：1.x の CSV 形式は可能な限り維持
  ・監査：共同編集時は lastModifiedBy / updatedAt を記録
  ・セキュリティ：公開 GitHub Pages 運用時はプロジェクトファイルを
    リポジトリに含めない（共有フォルダ or 私的ホスティング）
