■ 1. 製品概要
--------------------------------------------------------------------------------
PBPM（OmniStep）は、業務テーマ（親タスク）と子タスクを一覧・タイムラインで
管理する個人向けタスク管理ツールである。

  ・ブラウザのみで動作（HTML + CSS + JavaScript の静的構成）
  ・GitHub Pages 等への配置で URL から利用可能
  ・データは原則ブラウザ localStorage に保存（サーバー不要）
  ・CSV によるバックアップ・他担当者データの参照（外部CSV）に対応

想定ユーザー：個人の業務計画・進捗管理。小規模チームでは CSV 経由で
他メンバーの計画を「閲覧」できるが、共同編集機能は現時点では未実装。

■ 3. データ保存（localStorage）
--------------------------------------------------------------------------------
  キー名                    内容
  ------------------------  --------------------------------------------------
  omniStepData              自タスク一覧（JSON 配列）
  omnistep_master           年間日程マスター（JSON 配列）
  pbpm_project_title        表題（ヘッダー表示名）
  pbpm_owner_id             担当者ID（6桁・0埋め）
  pbpm_owner_name           担当者表示名
  pbpm_category_slots       カテゴリスロット設定
  pbpm_timeline_month_range タイムライン表示月数（3/6/9/12）
  pbpm_help_hover           ヘルプホバー ON/OFF
  pbpm_dark_mode            ダークモード ON/OFF
  pbpm_light_theme          ライト配色（sharp/cool/sepia/soft）

  保存方式：編集のたびに自動 save()（手動保存ボタン・終了警告は未実装）
  外部CSV取込データ：メモリ上のみ（localStorage には保存しない・閲覧専用）


■ 4. タスクデータモデル
--------------------------------------------------------------------------------
4.1 親子構造
  ・1テーマ＝1親タスク ＋ 0件以上の子タスク（枝番 010, 020, 030 …）
  ・親：テーマ名（PrimaryStep）、カテゴリ、テーマ色、集計表示
  ・子：業務内容（SecondaryStep）、着手日・納期・進捗・メモ等

4.2 主要フィールド（タスク1件）
  項目（CSV列名）     内部           説明
  ------------------  -------------  ----------------------------------------
  OwnerID             （設定値）     CSV入出力時の担当者識別（6桁）
  タスクID            taskCode       例：A010, A010-020
  カテゴリ            category       タブでフィルタ
  PrimaryStep         primaryStep    親：テーマ名
  FirstFlag / 展開日  firstFlag/msDate  ◇マイルストーン（年間マスター連動）
  TargetFlag/Date     targetName/    ☆イベント目標（年間マスター連動）
                      targetDate
  SecondaryStep       secondaryStep  子：業務内容
  着手日              startDate
  納期                deadline
  進捗                status         未着手/調査中/進行中/修正中/完了/中止
  メモ                memo
  アーカイブ済        archived       完了履歴タブへ
  fFlag               fFlag          依存連携（直前子の納期を下限に）
  テーマ色            themeAccentColor  親のみ
  進捗更新日          progressUpdatedAt  子・進捗変更時
  実績着手/納期       pdcaActualStart/  タイムライン PDCA 実績
                      pdcaActualEnd

4.3 年間日程マスター（1.2.0〜）
  ・name        名称
  ・date        開始日（単日の場合はこれのみ）
  ・dateEnd      終了日（任意・期間指定）
  ・isHoliday    休日フラグ（ON でタイムライン日曜帯と同表示）
  ・休日行は展開・イベント候補ドロップダウンから除外
  ・CSV：名称,日付,終了日,休日（旧2列CSVも取込可）


■ 5. 画面構成・主要機能
--------------------------------------------------------------------------------
5.1 表示モード
  ・リスト表示 … 表形式のタスク一覧（メイン）
  ・タイムライン表示 … ガント風の期間バー
  ・完了履歴タブ … archived タスク
  ・カテゴリタブ … 設定可能なカテゴリで絞込

5.2 リスト表示
  ・親行：チェック（CSV一括）、アコーディオン、テーマ色、カテゴリ、
          テーマ名、進捗集計、子追加ボタン
  ・子行：順序▲▼、業務内容、着手/納期、展開◇/イベント☆、進捗、
          依存連携 fFlag、メモ・中止・削除
  ・期限切れ行 … 背景・文字色で強調（expired）
  ・遅れタスクを含むテーマ … 親行上端・最下子行下端に赤太線（1.2.0）
  ・テーマ単位検索 … 一致テーマの親子をまとめて表示（アコーディオン展開）
  ・アコーディオン一括開閉（リスト）
  ・子並べ替え … FLIP アニメーション後に再描画

5.3 タイムライン表示
  ・表示範囲 … 今日の月から 3/6/9/12 か月（設定保持）
  ・月見出し … 折りたたみ可。交互色（白/薄灰）・区切り線強調（1.2.0）
  ・過去月 … 手動開閉可。未来の表示範囲外月は強制折りたたみ
  ・日曜 … 薄赤縦帯。土曜 … 点線。年間マスター休日も日曜同帯（1.2.0）
  ・今日 … 赤縦線
  ・親バー … 子期間の集約（ドラッグ不可・閲覧）
  ・子バー PDCA（B1・1.1.4） …
      計画：従来幅の破線枠（着手〜納期）
      実績：下段の実線（重ね表示）
      実績修正 OFF … 計画ごと移動・右端で納期伸縮
      実績修正 ON  … 実績のみ移動・右端で実績終了日伸縮
  ・◇マイルストーン / ★イベントマーカー
  ・依存線（fFlag 連携子間）
  ・遅れテーマ … 親上・最下子下に赤太線（一覧と共通ロジック）
  ・テーマ単位検索（リストと同様）

5.4 進捗・ステータス
  ・ステータス：未着手 → 調査中 → 進行中 → 修正中 → 完了（循環）
  ・中止 … 完了と同様にロック（進捗・メモ以外編集制限）
  ・親ステータス … 全子が終端（完了/中止）のとき、最後の子の状態を反映
  ・進捗シグナル（子） … ⚠ 着手遅れ / 🚀 進行良好 / ✨ 期限内完了 等
  ・親の中止 … 子も一括中止

5.5 年間日程（マスター）
  ・展開（FirstFlag）・イベント（Target）の名称・日付候補
  ・単日／期間・休日登録
  ・マスター CSV 入出力
  ・日程のみクリア / 全件削除

5.6 CSV・テンプレート
  ・CSV出力 … 親行チェックしたテーマ単位。OwnerID 必須
  ・CSV取込 … 自データへのマージ（OwnerID 一致行）
  ・外部CSV取込 … 他担当者データを閲覧専用で一覧・TLに併記
  ・外部担当者フィルタ … 併示 / 外部のみ / 担当者別
  ・外部タスクID表示 … （担当者ID）******
  ・テンプレート保存・読込 … 構成のひな形（別名展開）

5.7 設定・その他
  ・表題・担当者ID/名・カテゴリスロット（表示/非表示・名称）
  ・配色4種 + ダークモード
  ・版履歴モーダル（PBPM_VERSION_HISTORY 内蔵）
  ・チュートリアル・操作手順書（OmniStep_Manual.html）
  ・テーマ期間カレンダー（親期間・テーマ色）
  ・ヘルプホバー（？）


■ 6. 権限・編集制限（現行）
--------------------------------------------------------------------------------
  対象                    編集可否
  ----------------------  --------------------------------------------------
  自タスク（通常）        全項目編集可
  完了/中止/アーカイブ    進捗・メモ以外ロック
  外部CSVタスク           閲覧のみ（🔒）
  親タスク（TL）          バードラッグ不可
  子010                   依存連携 fFlag 対象外

■ 8. 非機能・制約
--------------------------------------------------------------------------------
  ・オフライン動作可（静的ファイル + localStorage）
  ・同時編集・競合解決なし（1ブラウザ1ユーザー想定）
  ・データ容量 … localStorage 上限（ブラウザ依存、おおよそ数MB）
  ・Access / SQLite 等の DB 直接接続なし
  ・ファイルパス指定によるプロジェクト保存なし
  ・認証・ユーザー管理なし（担当者IDは識別用のみ）
