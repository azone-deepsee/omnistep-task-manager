Aパート：基本操作
アプリを初めて使うとき。IDが未設定だとCSVの入出力で自他の区別ができません。
  1. 画面右上の ⚙（設定）ボタンをクリック
  2. 「担当者ID（6桁）」に自分の番号を入力（例：001234）
  3. 「担当者」に表示用の名前を入力
  4. 保存 をクリック
6桁に満たない場合は先頭にゼロが自動で付きます（例：123 → 000123）。
IDを後から変えると、過去に出力したCSVとの紐付けがずれます。最初に決めたら変えないのが安全です。
新しい業務テーマやプロジェクトを始めるとき。
  1. ＋新規テーマ登録 ボタンをクリック
  2. カテゴリを選択（設定で名前・表示/非表示を変更可能）
  3. テーマ名を入力（必須。ここが親タスクの名称になる）
  4. 必要に応じて着手日・完了日・展開（◇）・イベント（☆）を設定
  5. 「最初の子タスク」欄に1件目の子の内容を入力（任意）
  6. 登録する をクリック
最初の子タスクを入れると、着手日・完了日はその子に付きます。子が空なら親に直接付きます。
テーマの下に具体的な作業工程を追加するとき。
  1. リスト表示で、追加したいテーマの親行を見つける
  2. テーマ/課題列の ＋子タスク追加 ボタンをクリック
  3. 新しい行が末尾に追加されるので、業務内容を入力
  4. 着手日・納期・進捗を必要に応じて設定
子タスクは親と同じカテゴリを自動で継承します。親のカテゴリを変更すると子もすべて変わります。
タスクのスケジュールを設定・変更するとき。
  1. リスト表示の「着手」列で日付をクリック → カレンダーから選択
  2. 「納期」列で同様にクリック → カレンダーから選択
  3. タイムラインでもバーをドラッグして期間を移動できる（子タスクのみ）
  4. バー右端のハンドルをドラッグすると納期だけ伸縮する
着手日を入れると、納期が空なら同じ日に自動設定されます。納期を着手日より前にはできません。タイムラインのドラッグは計画のみ動き、実績は独立です（→ B-03）。
期限を過ぎると行が赤くなります（期限切れ表示）。進捗を「完了」にすれば解除されます。
棚卸・連休・発表会など全テーマ共通の基準日を管理するとき。
  1. ヘッダーの 📅 年間日程 ボタンをクリック
  2. 「名称」「開始日」「終了日」（任意）を入力。単日なら終了日は空のままでよい（終了日欄はグレー表示）
  3. 休日にしたい期間は 休日 にチェックを入れる
  4. 追加 する。一覧では名称・日付・休日をインライン編集できる
  5. タスクの「展開（◇）」や「イベント（☆）」のドロップダウンにマスターの候補が表示される
マスターの日付を変更すると、タスク側で同じ名称を選択している◇/★の日付も連動して変わります。
休日フラグ付きの日程は、タイムライン上で日曜と同じ帯として表示されます（連休の可視化に便利）。
例：「棚卸」=9/30 を登録 → テーマAの☆に「棚卸」を選択 → 自動的に9/30が入る。後で「棚卸」を10/3に変更 → テーマAの☆も10/3に連動。
全テーマの期間をガント風に俯瞰・調整するとき。
  1. ビュー切替の 📅 タイムライン ボタンをクリック
  2. 左側にテーマ・子タスクの名前、右側に期間バーが表示される
  3. 表示範囲を 3か月〜12か月 で切り替え
  4. 月ヘッダーの ＋/− で月ごとに折りたたみ可能（月見出しは交互色で区切りが分かりやすい）
  5. 子バーは破線枠＝計画、下段の実線＝実績（PDCA重ね表示）
  6. 単体移動／全体移動トグルで、子の個別移動と親によるテーマ一括スライドを切り替え（→ B-16）
  7. バーの上にホバーするとツールチップが表示される
赤い縦線が「今日」。日曜・休日マスターは薄い帯、土曜は点線で区別されます。親バーは子の集約表示で、直接ドラッグしません。
納期遅れのテーマは、一覧・タイムラインで親上端〜最下子下端に赤い太枠が付きます。
タイムラインで子バーをドラッグすると即座にデータが変更・保存されます。元に戻す機能はないため注意してください。
作業の進行状況を記録・更新するとき。
  1. 「進捗」列は表示のみ。左右の ◀ / ▶ で前後に切り替える
  2. 子タスクの順： 未着手 → 調査中 → 進行中 → 修正中 → 完了
  3. 親タスクは 進行中 ⇔ 修正中（未着手などからは ▶ で進行中へ）
  4. 未着手では ◀ が無効、完了では ▶ が無効、中止では両方無効
  5. 調査中から未着手へ戻す、または完了から戻すときは「○○の実績を差し戻しますか？」の確認が出る。OKなら実績データも削除される
初めて調査中以上に進めた日が実績の開始日になります（計画の着手日ではありません）。完了時は「実績開始〜完了日」が実績期間です。1日で未着手→完了ならその1日だけです。
親行に表示される「(2/5) 40%」は、子タスクの完了数と進捗率を自動計算した結果です。
中止は操作列の ⛔ から行います。中止中は進捗の ◀▶ は使えません。
タスクに詳細な備考や補足を残したいとき。
  1. 操作列の 🖋（メモ未入力）または 📝（メモあり）アイコンをクリック
  2. メモ入力ウィンドウが開くので、内容を入力
  3. 保存 をクリック → アイコンが 📝 に変わる
メモはCSV出力にも含まれます。改行はCSV内ではスペースに変換されます。
完了したテーマを一覧から整理するとき。
  1. テーマ内の子タスクをすべて「完了」または「中止」にする
  2. 親の進捗が自動で「完了」になったことを確認
  3. ツールバーの 履歴へ移動 をクリック
  4. 確認ダイアログで対象テーマを確認し、OK
  5. 「完了履歴」タブに移動される
履歴から戻したいときは「完了履歴」タブで、子タスクの ↺ ボタンをクリックすると親子ごと一覧に復帰します。
データのバックアップや他のスタッフとの共有をするとき。
  1. 出力したいテーマの親行に チェック を入れる（複数選択可）
  2. ツールバーの CSV出力 をクリック
  3. 確認ダイアログで対象を確認し、OK
  4. CSVファイルがダウンロードされる
CSVには担当者ID（OwnerID列）が含まれます。このIDで、取込み時に「自分の」「他人の」データが判別されます。
担当者IDが未設定だとCSV出力できません。先に A-01 を済ませてください。
バックアップの復元や、他スタッフのCSVを参照するとき。
  1. ツールバーの CSV取込 をクリック → ファイル選択が開く
  2. CSVファイルを選択
  3. OwnerIDが自分と一致する場合 → 本体データとして取り込み（重複があれば上書き確認）
  4. OwnerIDが自分と異なる場合 → 🔒外部CSV（閲覧専用）として表示
外部CSVは保存データには混ざりません。「外部表示をクリア」で消去できます。
旧形式（「内容」「課題」列）のCSVも自動判別して取り込めますが、新形式でのバックアップを推奨します。
毎年繰り返す業務の構成を雛形として残したいとき。

▼ テンプレート保存

  1. ヘッダーの テンプレート保存 をクリック
  2. テンプレート名を入力 → CSV がダウンロードされる

▼ テンプレート読込

  1. ヘッダーの テンプレート読込 をクリック
  2. ファイル選択でテンプレートCSVを開く
  3. グループ名（先頭に付く文字列）を入力 → 表題変更の確認
  4. 新規テーマとして一覧に追加される
テンプレートは日付・進捗をすべて空にした構成だけを保存します。読込時にマスター日程があれば◇/★の日付を自動補完します。
大量のタスクから特定のテーマや内容を探すとき。
  1. リスト表示の上部にある検索ボックスにキーワードを入力
  2. テーマ名・子タスク名・カテゴリ・メモなどから一致するものを検索
  3. ヒットした子タスクを含むテーマ全体（親＋全子）が表示される
検索はリスト表示・タイムライン表示の両方に反映されます。検索中はチェックボックスの一括操作が無効になります。
テーマが多いときに子タスクを折りたたんで見通しをよくしたいとき。
  1. テーマの － ボタンをクリック → 子タスクが非表示になる
  2. ＋ をクリック → 再び表示される
  3. 列ヘッダーの －（一括閉じ）で全テーマを一度に折りたたみ
リスト表示とタイムライン表示で開閉状態は独立しています。それぞれ別々に操作できます。
「タスクBはタスクAが終わるまで着手できない」という順序関係を設定するとき。
  1. 子タスク行のテーマ/課題列の右端にあるチェックボックスをON
  2. 直前の子タスクの納期が、この子の着手日の下限になる
  3. タイムラインでは紫色の依存線で前後関係が見える
  4. 前のタスクの納期を後ろへ動かすと、連動してこの子も押し出される
親行の 連携一括ON / OFF ボタンで、テーマ内の全子タスクの連携をまとめて切り替えられます。
最初の子タスク（枝番010）は依存の対象外です。また、完了済みタスクの連携は変更できません。
画面上のボタンや項目の意味を確認したいとき。
  1. 画面右上の ? ボタンをクリック（ON/OFFトグル）
  2. ONの状態で各ボタンや入力欄にマウスを乗せると、「名称：説明」形式のツールチップが出る
  3. もう一度 ? をクリックすると OFF
ヘルプ表示はON/OFFの状態がブラウザに記憶されます。操作に慣れたらOFFにすると邪魔になりません。
Bパート：裏技事典
親テーマと子タスクの両方の期間がすべて空の状態で、親に着手日を入力したとき。
  1. 親テーマ・子タスクの着手日と納期がすべて空であることを確認
  2. 親の「着手」列に日付を入力する（例：6/10）
  3. 自動的に子タスクへ 6/10、6/11、6/12… と1日ずつずれた日付が入る
  4. トースト「子タスクの着手日・納期を、親の着手日から1日ずつずらして設定しました」が表示される
子タスクが5つなら 6/10〜6/14 が自動で入ります。後からタイムラインでバーを個別に調整してください。
親または子にすでに日付が入っている場合は発動しません。完全に空の状態でのみ動作します。
子タスクの順序を ▲▼ で入れ替えたとき。
  1. 子タスクの行左下にある ▲ / ▼ をクリック
  2. 隣の子タスクと順番が入れ替わってアニメーションが表示される
  3. 入れ替わった2件の依存フラグ（fFlag）が自動でOFFになる
並べ替えで前後関係が変わると、依存の「前のタスクの納期が後ろの着手日の下限」ルールが矛盾する可能性があるため、安全のため自動解除されます。
タイムラインで子タスクの計画（破線枠）だけを動かしたいとき。
  1. タイムラインで子タスクのバーをドラッグして移動する（実績修正OFF時）
  2. 変わるのは計画（着手日・納期）だけ
  3. 実績（実績着手・実績納期）は動かない（ドラッグ中の見た目も固定）
実績だけを動かしたい場合は 実績修正 をONにしてください（→ B-12）。終了日を今日までまとめて伸ばすなら 実績更新（→ B-14）も使えます。
複数人のCSVを取り込んで、担当者ごとに表示を切り替えたいとき。
  1. 異なるOwnerIDのCSVを CSV取込 で複数回読み込む
  2. ヘッダーに「外部CSV表示フィルタ」のドロップダウンが出現する
  3. 「自分＋全外部」「外部のみ全担当」「特定の担当者のみ」を選択して切り替え
外部CSVの担当者IDは行のタスクID欄に「（ID）******」形式で表示されます。
テーマ名やアプリのタイトルを素早く変更したいとき。
  1. テーマ名：リスト表示のテーマ名テキストを直接クリック → その場で編集 → フォーカスを外すと保存
  2. アプリ表題：ヘッダーのタイトル文字を直接クリック → 編集 → フォーカスを外すと保存
アプリ表題を空にするとデフォルト名「Personal Business Project Manager」に戻ります。設定画面の「表題」欄でも変更できます。
完了・アーカイブ済み・外部CSVのテーマ名は編集できません（contenteditable が無効）。
テーマ内の子タスクをすべて「完了」や「中止」にしたとき。
  1. テーマ内のすべての子タスクを「完了」にする
  2. → 親の進捗表示が自動で 「完了」 に変わる
  3. 最後の子が「中止」の場合 → 親も 「中止」 になる
  4. 一部の子を「未着手」「進行中」などに戻すと → 親は 「進行中」 に戻る
判定基準は「枝番が最大の子タスクの状態」です。例えば子が5つあり、最後の子が「中止」なら親は「中止」になります。
子タスクの◇（展開）や★（イベント）の日付が着手日〜納期の範囲外になったとき。
  1. 子タスクに着手日=6/10、納期=6/20 を設定
  2. ◇の日付を 6/5（着手日より前）に入力しようとする
  3. → トースト「◇/★ はこの子タスクの期間内に設定してください」が表示され、入力が反映されない
すでにセット済みの◇/★がある状態で期間を狭めた場合は「◇が期間外です」と警告が出ます。自動で消えないので手動で修正してください。
テンプレートを読み込んだとき、すでに同じテーマ名が存在する場合。
  1. テンプレートを読み込むと、既存テーマと名前が一致するか自動チェック
  2. 一致した場合、ダイアログで3つの選択肢が表示される：
1: 追加 — 同名テーマがもう1セット作られる（別コード）
2: 上書き — 既存の親子を削除してから新しく取り込む
3: スキップ — このテーマは取り込まない
上書きを選ぶと既存データが完全に消えます。CSV出力でバックアップを取ってから操作してください。
テーマ内の日付をすべてクリアしてスケジュールを引き直したいとき。
  1. 親テーマの行で 期間リセット ボタンをクリック
  2. 確認ダイアログが表示される
  3. OKを押すと、このテーマの親・子すべての着手日と納期がクリアされる
完了済みやアーカイブ済みの子タスクはリセット対象外です。外部CSVもリセットされません。
リセット後に親の着手日を入れると、B-01 の自動連番が発動して子に1日ずつ入ります。
テーマごとに色を付けて視認性を上げたいとき。
  1. リスト表示で、親行の左端にある縦のカラーバーをクリック
  2. 10色のカラーパレットが表示される
  3. 好きな色をクリック → バーとタイムラインのバーに反映
  4. 「色を消す」で元のデフォルト色に戻る
テーマ色はCSV出力にも保存されるため、取り込んでも復元されます。
子タスクの横やタイムラインに記号が出たとき。

以下のアイコンは、期間と進捗状態を比較して自動的に表示されます。

記号意味表示条件
⚠️遅れの可能性経過日数に対して進捗ステップが追いついていない
🚀先行している経過日数に対して進捗が14%以上先に進んでいる
✨期限内完了納期より前に完了が記録された
✕中止進捗が「中止」のタスク
これらは自動計算のため手動で付け外しはできません。進捗を更新すれば変化します。
タイムラインで実績の期間だけを計画と独立して修正したいとき。
  1. タイムライン表示に切り替える
  2. 実績修正 ボタンをクリックしてONにする（ボタンがハイライト表示）
  3. 子タスクの実績バー（下段の実線）だけがドラッグ可能になる
  4. バーをドラッグ → 実績の着手日と終了日が移動
  5. 右端のハンドルをドラッグ → 実績の終了日だけ伸縮
  6. リスト表示に戻すと自動でOFFになる
実績修正OFF時は計画の破線枠だけが動きます。ON時は実績だけが動きます。「予定はこう、実際はこうだった」の比較用です。
納期を過ぎた実績部分は、タイムライン上で薄赤で表示されます。
未着手の子タスクには実績バーは出ません。◀▶で調査中以上に進めると、その日を起点に実績が付きます。
月間のテーマ重なりをカレンダー形式で確認したいとき。
  1. ヘッダー右側の今日の日付（例：「2026年7月18日(土)」）をクリック
  2. テーマ期間カレンダーがモーダルで開く
  3. ◀ / ▶ で月を切り替え
  4. 各テーマの期間が週単位のバーで色分け表示される
このカレンダーは閲覧専用です。タイムラインのように直接編集はできませんが、「この月はテーマが重なりすぎていないか」の確認に便利です。
調査中・進行中・修正中の子タスクの実績終了日を、まとめて今日まで進めたいとき。
  1. リスト表示またはタイムライン表示で 実績更新 をクリック
  2. 対象ステータスの子タスクについて、実績終了日が今日まで更新・保存される
  3. すでに実績着手がある場合は着手日はそのまま、終了だけ今日へ
  4. 実績がまだ無い場合は今日単日でシードされる（計画着手日には寄せない）
アプリ起動時にも同じ処理が静かに走ります。進捗を調査中以上へ進めたときも、そのタスクだけ実績が今日に同期されます。
完了・中止・未着手・外部CSV行は対象外です。
アプリのバージョンと変更内容を確認したいとき。
  1. 画面右上の ⚙（設定）を開く
  2. 「PBPM バージョン x.x.x」のリンクをクリック
  3. 版履歴モーダルが開く。各 ver はアコーディオン（クリックで開閉）
  4. 最新版だけ最初から開いており、一覧はモーダル内でスクロールできる
版履歴はアプリ内蔵データです。localStorage には依存しません。
テーマ（プロジェクト）の計画を、子の相対関係を保ったまま前後にずらしたいとき。
  1. タイムライン表示で 単体移動 トグルをクリック → 全体移動（ハイライト）にする
  2. 親バーをドラッグすると、テーマ内のすべての子の計画（着手・納期）が同じ日数だけ平行移動する
  3. 親の期間は従来どおり子の集約で再計算される
  4. リスト表示に戻すと自動で 単体移動 に戻る
◇／★・実績・依存フラグ（fFlag）は動かしません。子同士の間隔と依存関係はそのまま維持されます。
子に調査中・進行中・修正中・完了・中止が1件でもあるテーマはスライドできません。親を動かそうとすると「実行中のタスクがあるプロジェクトは動かせません。」と表示されます。
全体移動中は子バーの個別ドラッグ／納期伸縮はできません。実績修正とは同時に使えません（片方をONにすると他方はOFF）。
直前のタスク変更を取り消したいとき。
  1. リストまたはタイムラインの 戻る をクリック（または Ctrl+Z / ⌘Z）
  2. 直前に保存されたタスク状態へ戻る
タイムラインのドラッグは「ドラッグ開始前」が1手分です。途中の細かい移動はまとめて取り消せます。
戻れるのは1手だけです。やり直し（Redo）はありません。ページを再読み込みすると履歴は消えます。年間マスターや設定の変更は対象外です。
